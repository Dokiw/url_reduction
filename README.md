**Сокращатель ссылок,содержит 2 эндпоинта, согласно тз.**

ограничения: 
Технологии 
● Python 3.12+ 
● git 
● sqlite3 (стандартная библиотека, без ORM, было грустно, но терпимо) 
● pytest 
● Фреймворк для api на выбор (выбран fastapi)

**Запуск:**
Скачивание проекта
git clone https://github.com/Dokiw/url_reduction.git
cd url_reduction

активируем окружение:
python -m venv .venv
source .venv/bin/activate   # Linux/Mac
.venv\Scripts\activate # Windows

установка зависимостей:
pip install -r requirements.txt

и запускаем приложение: 
uvicorn main:app --reload

запуск тестов:
pytest
покрыты основные фукции: 
  репозиторий
  сервис
  и замокан репозиторий

логирование:
Все события создания короткой ссылки логируются через logger (из config).
404 при попытке получить несуществующую ссылку также логируется как предупреждение.
Логи помогают отслеживать успешные и неуспешные операции, что важно для мониторинга сервиса.

**Комментарии по проекту**

По поводу архитектуры:
При своих технологиях я старался сделать проект чистым и близким к solid разделение уровней отвественности, 
router - service - repo, не мог применить паттерны uow и т.д, потому что мы работаем с прямыми сырыми запросами.
Также, я бы добавил паттерн интерфейсов и передачи зависимостей, чтоб добиться независимости реализации service и repo, чтоб добиться 
более чистой архитектуры

в дальнейшем важным моментом будет, перевод в ассинхронный вариант работы, но т.к мы используем sqlite3, а не ассинхронный вариант, 
то смысла делать код ассинхронным почти небыло, мы бы не добились прироста в скорости, т.к запросы в бд будут sync. 

также, имеет смысл добавления redis и использование индексации бд, возможно, использование orm обертки, в целом, в скорости не потеряем, 
но появятся более удобные методы управления и контроля данных, в том числе и полноценная атомарность.

полноценные комиты не получилось делать, т.к делал на локале, поэтому проект вышел, как есть, но я бы отменял каждый пункт по добавлению, 
условно, начал с пустых методов(без реализации), а после, добавление уже реализации с покрытием тестами.

Минусы:
1. синхронность(думаю, очевидно)
2. нету фильтров и параметров по проекту
Плюсы:
1. простота реализации кода
2. чистота в проекте, можно легко расщирять.
